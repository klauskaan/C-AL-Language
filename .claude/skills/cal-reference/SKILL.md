# C/AL Language Reference

## Supported Versions (Role Tailored Client)

C/AL, as referenced here, runs in the Role Tailored Client (RTC). Earlier "classic client" versions (NAV 2009 and before) are out of scope.

| Version | Name | Platform Number | Notes |
|---------|------|-----------------|-------|
| NAV 2013 | Dynamics NAV 2013 | 7.0 | First RTC-only version |
| NAV 2013 R2 | Dynamics NAV 2013 R2 | 7.1 | |
| NAV 2015 | Dynamics NAV 2015 | 8.0 | |
| NAV 2016 | Dynamics NAV 2016 | 9.0 | Added FOREACH, .NET events |
| NAV 2017 | Dynamics NAV 2017 | 10.0 | |
| NAV 2018 | Dynamics NAV 2018 | 11.0 | Last version branded "NAV" |
| BC13 | Business Central October 2018 | 13.0 | C/AL + AL extensions side by side |
| BC14 | Business Central April 2019 | 14.0 | Last version supporting C/AL |

Version 12.0 was never publicly released. BC15 (15.0) dropped C/AL entirely.

## @ Numbering System

All variables and procedures have unique `@` numbers (auto-generated by C/SIDE):

```cal
VAR
  Text001@1000 : TextConst 'ENU=Message text';
  Customer@1001 : Record 18;
  Amount@1002 : Decimal;

PROCEDURE Calculate@1(Param@1000 : Integer) : Decimal;
```

## Variable Modifiers (NAV 2013+)

### SECURITYFILTERING (NAV 2013 R2+)

Controls security filtering behavior for Record and Query variables.

**As a variable modifier:**
```cal
VAR
  User@1003 : Record 2000000120 SECURITYFILTERING(Filtered);
  CustQuery@1000 : Query 21 SECURITYFILTERING(Filtered);
  LogEntry@1000 : Record 5065 SECURITYFILTERING(Ignored);
```

**Valid values:**
- `Filtered` - Apply security filters (enforces record-level security)
- `Ignored` - Bypass security filters (use with caution)
- `Validated` - Apply validation (documented but rarely used)
- `Disallowed` - Prevent access (documented but rarely used)

**As a property/method on RecordRef (runtime control):**
```cal
// Property assignment
RecRef."SECURITYFILTERING" := SECURITYFILTER::Filtered;

// Method call on Record variable
CustomerSalesYTD."SECURITYFILTERING"(SECURITYFILTER::Filtered);
```

**SECURITYFILTER enum values:** `SECURITYFILTER::Filtered`, `SECURITYFILTER::Ignored`, `SECURITYFILTER::Validated`, `SECURITYFILTER::Disallowed`

### Other Variable Modifiers

Modifier order after data type: `TEMPORARY` → `INDATASET` → `WITHEVENTS` → `RUNONCLIENT` → `SECURITYFILTERING`

**Note:** When both WITHEVENTS and RUNONCLIENT are present, NAV always exports them as `WITHEVENTS RUNONCLIENT` (31 occurrences found in real NAV exports, 0 in reversed order).

```cal
VAR
  TempCustomer@1000 : TEMPORARY Record 18;           // In-memory only
  ShowAmount@1001 : Boolean INDATASET;               // Page variable binding
  DotNetVar@1002 : DotNet "..." RUNONCLIENT;         // Client-side .NET
  AutoVar@1003 : Automation "..." WITHEVENTS;        // Event handling
```

## Procedure Syntax

```cal
[External]
PROCEDURE Name@1(VAR Param@1000 : Type) : ReturnType;
VAR
  LocalVar@1001 : Type;
BEGIN
  // Code here
END;

LOCAL PROCEDURE PrivateFunc@2();
BEGIN
  // Private procedure
END;
```

- `[External]` - Public API attribute
- `LOCAL` - Private procedure
- `VAR` - Pass by reference
- `@` numbers - Unique identifiers

## TextConst (Multi-Language Strings)

```cal
Text001@1000 : TextConst 'ENU=English;FRA=French;DEU=German';
```

Language codes: `ENU` (English US), `FRA` (French), `DEU` (German), etc.

## Literals

### Strings
Single quotes only: `'text here'`
Escaped apostrophe: `''` (double single quote)

### Numbers
- Integer: `123`, `-456`
- Decimal: `123.45`
- Hexadecimal: `0x1A2B` (NAV 2013+)

### Dates and Times
- Date: `MMDDYY[YY]D` - Example: `060120D` = June 1, 2020
- Undefined date: `0D`
- Time: `HHMMSS[MS]T` - Example: `120000T` = 12:00:00
- Undefined time: `0T`
- DateTime: `MMDDYY[YY]DHHMMSS[MS]T` - Example: `060120D120000T`
- Undefined DateTime: `0DT`

### Boolean
`TRUE`, `FALSE`

### Option Values
Use `::` syntax: `"Account Type"::Customer`, `Status::Open`

## Comments

```cal
// Single-line comment (RECOMMENDED)

{ Multi-line comment
  Traditional NAV style }

/* Multi-line comment
   C-style (NAV 2013+) */
```

**Warning**: `{ }` comments only work inside `BEGIN...END` blocks. In FIELDS/KEYS/CONTROLS sections, `{ }` are structural delimiters.

## .NET Interoperability (NAV 2016+)

```cal
VAR
  MyDotNet@1000 : DotNet "'Assembly'.Type" WITHEVENTS;

// FOREACH loop
FOREACH Item IN Collection DO
  ProcessItem(Item);

// Event trigger
MyDotNet::EventName(sender : Variant; e : DotNet "'System'.EventArgs");
BEGIN
  // Handle event
END;
```

### Syntax Differences: C/AL vs AL

**Event Declaration:**
```cal
// C/AL (correct for this extension)
PROCEDURE MyEvent@1000();
INTERNALEVENT
BEGIN
END;
```
```al
// AL (DO NOT support)
[IntegrationEvent(false, false)]
procedure MyEvent()
begin
end;
```

**Case Convention:**
- C/AL: UPPERCASE keywords (`BEGIN`, `IF`, `THEN`)
- AL: lowercase keywords (`begin`, `if`, `then`)

## Commonly Confused Features

Sometimes confused as AL-only but actually valid in C/AL:

| Feature | Status | Notes |
|---------|--------|-------|
| Compound operators `+=`, `-=`, `*=`, `/=` | C/AL (all versions) | Found in NAV 2018 standard code |
| `FOREACH` loop | C/AL (NAV 2016+) | .NET collection iteration |
| `EVENT`, `WITHEVENTS` | C/AL (NAV 2016+) | .NET event subscriptions |
| `DotNet` type | C/AL (all supported versions) | .NET interop |
| `[External]` attribute | C/AL | Public API marker |
